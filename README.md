## 협업 규칙
### 소통 채널
- 매일 오전 10시 반 줌으로 미팅(수업 있는 날, 오후 2시)
### 브랜치 전략
- 기능별 브랜치 생성(ex. be-feat-oauth)
- 기능 개발 완료에 따라 분야별 브랜치에 병합(ex. be-feat-oauth -> be-dev)
- 코드리뷰 완료에 따라 대표 브랜치에 병합(ex. be-dev -> team-2)
### 커밋 전략
